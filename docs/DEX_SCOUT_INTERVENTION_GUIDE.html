Architecture

Runtime roles

  • UI / Streamlit: app.py compatibility shell with ui/pages_*.py render modules; app.py is now compatibility shell, not business owner.
  • Worker: worker.py delegates through app_runtime_facade.py.
  • Webhook: tg_webhook.py delegates through app_runtime_facade.py.
  • Dash/read-only sidecar: may inspect state but must not mutate runtime state.

Persistence

  • Primary: Cloudflare D1 through D1 Worker and app_storage / repository helpers.
  • Fallback: local CSV/text fallback remains supported where code paths allow it.
  • Legacy: Supabase references are legacy/fallback only, not the current primary persistence path.

Extracted service/core layer

  • runtime_core.py, storage_core.py, storage_repository.py
  • notification_core.py, monitoring_core.py, monitoring_service.py
  • portfolio_service.py, app_service.py, scanner_service.py, scanner_sources.py

Data Flow

  1. Runtime jobs resolve through app_runtime_facade.py so scan_cycle, monitor_cycle, and notification jobs keep stable signatures.
  2. scanner_sources.py owns source adapter result contracts for DexScreener search and injected Birdeye trending fetches; scanner_service.py owns deterministic normalization, filtering diagnostics, refill planning, cycle orchestration, and Live Pulse payload materialization.
  3. Live Pulse writes preserve live_pulse_candidates.json. Empty or failed scans should still write counters and last_empty_reason when possible.
  4. Monitoring Priority rows continue to use monitoring_core.py and monitoring_service.py; Live Pulse candidates must not override Priority watchlist materialization.

Playbook

  • Runtime matrix roles: ui_streamlit, worker, webhook, dash_readonly, app_compat, core_modules, and golden_fixtures.
  • Use no secrets in guide examples, logs, matrix output, screenshots, or PR notes.
  • Before merging service extraction PRs, run python scripts/runtime_no_fail_matrix.py --json runtime-no-fail-summary.json and pytest -q.
  • For scanner changes, verify scanner_sources.py and scanner_service.py import without streamlit or app.py and perform no source fetch, storage write, or Telegram send at import time.
  • After merge, run runtime smoke modes in order: runtime_matrix, read_only, monitor_cycle, notify_cycle, then scan_cycle.
  • Operator checklist: open the Runtime page, verify runtime_matrix/app_compat/golden_fixtures, check the Live Pulse empty/source reason before rerunning scan, and run monitor_cycle/notify_cycle before scan_cycle after an unstable deploy.
  • In Streamlit, check Scout Live Pulse, Monitoring Priority watchlist, and Runtime/debug diagnostics for visible counters and empty reasons.
